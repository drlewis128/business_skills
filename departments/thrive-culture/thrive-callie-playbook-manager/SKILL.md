# THRIVE Callie Playbook Manager

Callie is the culture program's content engine — she translates THRIVE principles into operational playbooks and skills that leaders and teams can actually use. This skill manages Callie's role, the skill development pipeline, and the coordination with Fitz Bowen's testing function, ensuring that new THRIVE content is high-quality, tested, and deployed on cadence.

## When to Use

- Dr. Lewis needs a current view of Callie's skill development pipeline and output
- Callie is proposing a new skill and needs the intake and review process documented
- Fitz has completed a pilot test and the results need to be incorporated into the pipeline record
- Matt Mathison is reviewing the annual THRIVE program and needs Callie + Fitz output summarized

## Output Contract

- Fitz's sign-off is a deployment gate — no THRIVE skill created by Callie reaches an entity team without a real-world pilot test, because skills that look correct in a document often fail in the hands of managers under time pressure, and a single bad deployment experience with an untested skill erodes trust in the entire THRIVE program infrastructure.

- Dr. Lewis's review at Stage 1 (intake) and Stage 2 (draft complete) creates two checkpoints before Fitz's time is spent on testing — catching a poorly-framed skill need at intake or a draft with canonical format errors before pilot saves weeks of rework and ensures Fitz's testing capacity is used on skills that are already structurally sound.

- Callie's quarterly output (3 skills target) is tracked as a program OKR (thrive-culture-okr-tracker KR4) and reported to Matt Mathison annually — making Callie's work visible in the OKR system is the mechanism by which culture content development is treated as a real program function with accountability, rather than as background administrative activity.

- THRIVE skill registry (current coverage; gap identification)
- thrive-skills-development-plan output (skill gap reports that feed Callie's backlog)
- thrive-exit-interview-analyzer (controllable departure patterns that signal playbook gaps)
- thrive-pulse-survey-builder anomaly flags (culture signal → playbook need)

### Writes to
- THRIVE skill registry (new skills added at Stage 4 deployment)
- thrive-culture-okr-tracker (KR4 coverage updates)
- Dr. Lewis quarterly pipeline report (Callie + Fitz output)
- Matt Mathison annual THRIVE program briefing (playbook section)

### HITL Required
- Dr. Lewis approves every skill intake (Stage 1) and every draft (Stage 2) before Fitz testing begins
- Fitz signs off in writing before any skill is deployed to an entity team
- Matt Mathison reviews annual playbook state in January briefing; no signature required but attendance is expected

## Test Cases

1. **Golden path:** Q2 pipeline review. Callie has 3 skills at Stage 4 (published), 1 in Stage 3 (Fitz testing), 2 in Stage 2 (drafting). Fitz cleared all 3 published skills with average pilot feedback of 4.1/5. OKR KR4 updated: 3 skills added. Pipeline health: On track. One new backlog item added from an exit interview signal (Enjoy gap in a HIVE team; no workload management skill exists). Added to backlog at High priority.

2. **Edge case:** Fitz pilots a new THRIVE recognition skill with an Allevio team and finds that the nomination language is confusing for clinical staff who are not familiar with management terminology. Returns skill to Callie (Stage 2) with specific revision notes. Callie revises in 5 days. Fitz re-pilots with 5-person cohort (abbreviated re-test). Cleared on second pass. Deployed in same quarter — still counts toward Callie's Q target.

3. **Adversarial:** An entity CEO requests that Callie develop a custom THRIVE skill for their entity without going through the Dr. Lewis review and Fitz testing pipeline. Callie escalates to Dr. Lewis. Dr. Lewis declines the bypass — all THRIVE skills go through the same intake, review, and test pipeline regardless of requestor seniority. Entity CEO's request is logged as a backlog item with High priority and normal pipeline processing begins. CEO is given an expected timeline.

## Audit Log

Pipeline records are maintained by Dr. Lewis and Callie as a joint program artifact. Every stage transition for every skill is timestamped (intake date, draft review date, Fitz test start/end, deployment date). Fitz's sign-off documents are retained as quality records. Skills that are returned to Stage 2 from Stage 3 are logged with Fitz's revision feedback summary. Annual playbook review records are retained as year-end program artifacts.

## Deprecation

This skill is deprecated when Callie's role evolves beyond individual playbook development to a broader culture function that requires a more complex skill management framework, or when MBL adopts a formal learning management system that provides pipeline tracking natively. The intake → review → test → deploy sequence is considered foundational to skill quality and must be preserved in any successor framework. Dr. Lewis must approve any change to the pipeline stage gates.
